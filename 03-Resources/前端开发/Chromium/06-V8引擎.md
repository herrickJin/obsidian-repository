## 🏗️ V8 架构概述

### V8 是什么？

V8 是 Google 开发的高性能 JavaScript 引擎，使用 C++ 编写，用于：
- Chromium / Chrome 浏览器
- Node.js 运行时
- Deno 运行时
- Electron 应用

## 🔍 解析器 (Parser)

### 解析过程

```javascript
// JavaScript 代码
function add(a, b) {
  return a + b
}

// Token 流 (词法分析)
// FUNCTION, IDENTIFIER(add), LPAREN, IDENTIFIER(a), COMMA,
// IDENTIFIER(b), RPAREN, LBRACE, RETURN, IDENTIFIER(a),
// PLUS, IDENTIFIER(b), RBRACE

// AST (语法分析)
{
  type: "FunctionDeclaration",
  id: { type: "Identifier", name: "add" },
  params: [
    { type: "Identifier", name: "a" },
    { type: "Identifier", name: "b" }
  ],
  body: {
    type: "BlockStatement",
    body: [{
      type: "ReturnStatement",
      argument: {
        type: "BinaryExpression",
        operator: "+",
        left: { type: "Identifier", name: "a" },
        right: { type: "Identifier", name: "b" }
      }
    }]
  }
}
```

### 优化技术

```javascript
// 1. 内联 (Inlining)
function add(a, b) {
  return a + b
}

function calculate(x, y) {
  return add(x, y) * 2  // add 函数会被内联
}

// 优化后相当于:
function calculate(x, y) {
  return (x + y) * 2  // 直接计算，无函数调用开销
}

// 2. 逃逸分析 (Escape Analysis)
function createPoint(x, y) {
  return { x, y }  // 对象逃逸 (被返回)
}

function sumPoint(x, y) {
  const p = { x, y }  // 对象不逃逸
  return p.x + p.y    // 可以优化为标量替换
}

// 优化后相当于:
function sumPoint(x, y) {
  return x + y  // 不创建对象
}

// 3. 循环优化
function sum(arr) {
  let total = 0
  for (let i = 0; i < arr.length; i++) {
    total += arr[i]
  }
  return total
}

// 优化: 边界检查消除、循环不变量外提
```

### 去优化 (Deoptimization)

```javascript
// 去优化触发条件

function process(value) {
  // 假设优化时 value 总是整数
  return value + 1
}

// 优化后的代码假设 value 是整数
process(1)      // ✅ 使用优化代码
process(2)      // ✅ 使用优化代码
process('hello') // ❌ 触发去优化，回退到字节码

// 去优化后的重新优化
// 如果后续调用恢复稳定，会重新触发优化
```

## 🎭 Hidden Classes

### 什么是 Hidden Classes？

Hidden Classes (隐藏类) 是 V8 用于优化对象属性访问的机制。

```
┌─────────────────────────────────────────────────────────────┐
│                    Hidden Classes                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  JavaScript 对象:                                           │
│  let p1 = { x: 1, y: 2 }                                   │
│  let p2 = { x: 3, y: 4 }                                   │
│                                                             │
│  共享相同的 Hidden Class:                                    │
│  ┌─────────────────────────┐                               │
│  │    Hidden Class A       │                               │
│  │  ┌─────┬─────┐         │                               │
│  │  │  x  │  y  │         │                               │
│  │  │ [0] │ [1] │ 偏移量   │                               │
│  │  └─────┴─────┘         │                               │
│  └─────────────────────────┘                               │
│                                                             │
│  属性访问: p1.x                                             │
│  1. 查找 Hidden Class                                       │
│  2. 获取属性偏移量 [0]                                       │
│  3. 直接内存访问                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Hidden Class 转换

```
┌─────────────────────────────────────────────────────────────┐
│                 Hidden Class 转换链                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  let obj = {}                                               │
│                                                             │
│  [HC0] 空 Hidden Class                                      │
│    │                                                        │
│    │ obj.x = 1                                              │
│    ▼                                                        │
│  [HC1] 添加属性 x                                           │
│    │                                                        │
│    │ obj.y = 2                                              │
│    ▼                                                        │
│  [HC2] 添加属性 y                                           │
│                                                             │
│  不同的添加顺序会产生不同的 Hidden Class:                     │
│                                                             │
│  obj1 = {}; obj1.x = 1; obj1.y = 2  → HC0 → HC1 → HC2      │
│  obj2 = {}; obj2.y = 1; obj2.x = 2  → HC0 → HC3 → HC4      │
│                                                             │
│  obj1 和 obj2 有不同的 Hidden Class!                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 最佳实践

```javascript
// ✅ 好: 保持对象形状一致
function Point(x, y) {
  this.x = x
  this.y = y
}

const p1 = new Point(1, 2)
const p2 = new Point(3, 4)  // 共享 Hidden Class

// ❌ 差: 动态添加属性
const p3 = new Point(5, 6)
p3.z = 7  // 创建新的 Hidden Class

// ✅ 好: 构造函数中初始化所有属性
function User(name, age) {
  this.name = name
  this.age = age
  this.active = true  // 即使有默认值也要初始化
}

// ❌ 差: 条件性添加属性
function User2(name, age, isAdmin) {
  this.name = name
  this.age = age
  if (isAdmin) {
    this.admin = true  // 不一致的形状
  }
}
```

## 🔄 Inline Caches (IC)

### IC 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    Inline Caches                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  属性访问: obj.x                                            │
│                                                             │
│  第一次访问:                                                 │
│  1. 查找 Hidden Class                                       │
│  2. 查找属性偏移量                                           │
│  3. 缓存结果: [HC, offset]                                  │
│                                                             │
│  后续访问:                                                   │
│  1. 检查 Hidden Class 是否匹配缓存                          │
│  2. 匹配: 直接使用缓存的偏移量                               │
│  3. 不匹配: 重新查找并更新缓存                               │
│                                                             │
│  IC 状态:                                                   │
│  ├── Uninitialized: 未初始化                                │
│  ├── Monomorphic: 单态 (一种类型)                           │
│  ├── Polymorphic: 多态 (2-4 种类型)                         │
│  └── Megamorphic: 超多态 (超过 4 种) - 性能下降              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### IC 优化示例

```javascript
// ✅ 好: 单态 IC
function getX(obj) {
  return obj.x  // IC 缓存一种 Hidden Class
}

const point = { x: 1, y: 2 }
getX(point)  // Monomorphic
getX(point)  // 命中缓存

// ❌ 差: 多态 IC
function getX2(obj) {
  return obj.x  // IC 需要处理多种形状
}

getX2({ x: 1, y: 2 })     // 形状 1
getX2({ x: 1, y: 2, z: 3 }) // 形状 2 - Polymorphic
getX2({ x: 1 })           // 形状 3 - Polymorphic
```

## 🗑️ 垃圾回收 (GC)

### 分代垃圾回收

```
│                                                             │

│  │  ┌─────────────┐ ┌─────────────┐                   │   │
│  │  │   From      │ │     To      │                   │   │

│                          │                                  │
│                          │ 晋升 (存活多次)                   │
│                          ▼                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Old Generation                      │   │
│  │  (老生代 - 大，较少 GC)                              │   │

│  │  │              Old Space                       │   │   │

│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Large Object Space                     │   │
│  │  (大对象空间 - > 256KB 的对象)                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### GC 算法

```
┌─────────────────────────────────────────────────────────────┐
│                     GC 算法                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Scavenge (新生代):                                         │
│  ├── 复制存活对象到 To Space                                │
│  ├── 交换 From 和 To                                        │
│  ├── 速度快，但只用一半空间                                  │
│  └── 适合短命对象                                            │
│                                                             │
│  Mark-Sweep-Compact (老生代):                               │
│  ├── Mark: 标记所有存活对象                                 │
│  ├── Sweep: 清除未标记对象                                  │
│  ├── Compact: 整理碎片                                     │
│  └── 会暂停执行 (Stop-The-World)                            │
│                                                             │
│  Orinoco (并发/增量 GC):                                    │
│  ├── 并行标记                                                │
│  ├── 增量标记                                                │
│  ├── 并发清扫                                                │
│  └── 减少主线程暂停时间                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 内存优化技巧

```javascript
// 1. 避免内存泄漏
// ❌ 未清理的事件监听
element.addEventListener('click', handler)
// 忘记移除

// ✅ 及时清理
element.removeEventListener('click', handler)

// 2. 避免闭包持有大对象
// ❌ 闭包持有大数组
function createHandler() {
  const largeData = new Array(1000000)
  return () => console.log(largeData.length)
}

// ✅ 只保留需要的值
function createHandler2() {
  const largeData = new Array(1000000)
  const length = largeData.length  // 只保存长度
  return () => console.log(length)
}

// 3. 使用 WeakMap/WeakSet
// ✅ 不阻止 GC
const cache = new WeakMap()
cache.set(obj, metadata)  // obj 可以被 GC

// 4. 及时释放引用
// ❌
let data = loadData()
// 使用 data...
// data 一直存在

// ✅
{
  let data = loadData()
  // 使用 data...
}  // data 离开作用域，可以被 GC
data = null  // 显式释放
```

```javascript
  global.gc()
}

// 性能测试
function benchmark(fn, iterations = 1000000) {
  const start = performance.now()
  for (let i = 0; i < iterations; i++) {
    fn()
  }
  const end = performance.now()
  console.log(`Time: ${end - start}ms`)
  console.log(`Per op: ${(end - start) / iterations}ms`)
}
```
