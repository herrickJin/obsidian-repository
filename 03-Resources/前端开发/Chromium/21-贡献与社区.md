## 👀 代码审查

### 审查流程

```
┌─────────────────────────────────────────────────────────────┐
│                    代码审查流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Gerrit 代码审查系统                                         │
│  网址: https://chromium-review.googlesource.com/            │
│                                                             │
│  流程:                                                       │
│  1. 提交 Change                                             │
│     └── git cl upload                                       │
│                                                             │
│  2. 自动检查                                                 │
│     ├── Presubmit 检查                                      │
│     ├── Try Jobs (编译测试)                                 │
│     └── 静态分析                                            │
│                                                             │
│  3. 代码审查                                                 │
│     ├── 自动分配审查者                                       │
│     ├── 审查者评审                                           │
│     └── 提出修改意见                                         │
│                                                             │
│  4. 修改和更新                                               │
│     ├── 修改代码                                            │
│     ├── git cl upload                                       │
│     └── 等待新审查                                          │
│                                                             │
│  5. 获得批准                                                 │
│     ├── LGTM (Looks Good To Me)                            │
│     ├── 代码所有者批准                                       │
│     └── 提交队列检查                                        │
│                                                             │
│  6. 提交                                                     │
│     └── git cl land                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 📤 提交 Patch

### 准备工作

```bash
# 1. 配置 Git
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 2. 配置 depot_tools
# 确保 depot_tools 在 PATH 中

# 3. 签署 CLA (Contributor License Agreement)
# 访问 https://cla.developers.google.com/

# 4. 认证
# 首次使用会提示登录 Google 账号
git cl creds
```

### 提交流程

```bash
# 1. 创建分支
git checkout -b my-feature origin/main

# 2. 修改代码
# 编辑文件...

# 3. 本地测试
ninja -C out/Debug chrome
ninja -C out/Debug unit_tests

# 4. 运行测试
out/Debug/unit_tests --gtest_filter="MyTest.*"

# 5. 提交到暂存区
git add .
git commit -m "Description of changes"

# 6. 上传到 Gerrit
git cl upload

# 7. 回应审查意见
# 修改代码后
git add .
git commit --amend
git cl upload

# 8. 提交 (获得批准后)
git cl land
```

## 📋 设计文档

### 提案流程

```
┌─────────────────────────────────────────────────────────────┐
│                    设计提案流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 创建设计文档                                             │
│     ├── 使用 Google Docs 或 Markdown                        │
│     ├── 使用模板                                            │
│     └── 放在 //docs/ 目录                                   │
│                                                             │
│  2. 请求审查                                                 │
│     ├── 发送到相关邮件列表                                   │
│     ├── 创建 Intent to Implement (I2I) issue               │
│     └── 收集反馈                                            │
│                                                             │
│  3. 讨论和修改                                               │
│     ├── 回应评论                                            │
│     ├── 修改设计                                            │
│     └── 达成共识                                            │
│                                                             │
│  4. 获得批准                                                 │
│     ├── API Owners 批准                                     │
│     ├── 安全审查                                            │
│     └── 隐私审查                                            │
│                                                             │
│  5. 实施                                                     │
│     ├── 编写代码                                            │
│     ├── 编写测试                                            │
│     └── 更新文档                                            │
│                                                             │
│  6. 发布                                                     │
│     ├── 功能标志                                            │
│     ├── 逐步推出                                            │
│     └── 监控和修复                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 设计文档模板

```markdown
# [功能名称]

## Summary
一句话描述功能

## Motivation
为什么要做这个功能

## Goals
- 目标 1
- 目标 2

## Non-Goals
- 不在范围内的事情

## Design

### Overview
整体设计概述

### API Design
接口设计，包括 IDL 定义

### Implementation
实现细节

### Security Considerations
安全考虑

### Privacy Considerations
隐私考虑

## Alternatives Considered
考虑过的替代方案

## Testing Plan
测试计划

## Timeline
时间线

## References
相关文档和链接
```
