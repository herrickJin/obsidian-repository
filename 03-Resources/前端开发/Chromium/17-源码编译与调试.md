## 🛠️ depot_tools

### 安装配置

```bash
# 1. 获取 depot_tools
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git

# 2. 添加到 PATH
export PATH="/path/to/depot_tools:$PATH"

# 3. 添加到 shell 配置 (~/.bashrc 或 ~/.zshrc)
echo 'export PATH="/path/to/depot_tools:$PATH"' >> ~/.zshrc

# 4. 更新 depot_tools
# depot_tools 会自动更新
# 或手动: gclient

# 5. Windows 特殊配置
# 需要安装 Visual Studio 2022
# 设置 DEPOT_TOOLS_WIN_TOOLCHAIN=0 使用本地 VS
```

## 📥 获取源码

### 下载 Chromium

```bash
# 1. 创建目录
mkdir chromium && cd chromium

# 2. 下载完整源码 (约 40GB，需要科学上网)
fetch --nohooks chromium

# 3. 进入源码目录
cd src

# 4. 安装构建依赖
# Linux
./build/install-build-deps.sh

# macOS
# 需要安装 Xcode 和 Command Line Tools

# 5. 运行钩子
gclient runhooks

# 6. 更新源码
git pull origin main
gclient sync
```

## 🔨 GN 构建系统

### 生成构建文件

```bash
# 生成 Debug 构建
gn gen out/Debug

# 生成 Release 构建
gn gen out/Release --args="is_debug=false"

# 常用参数
gn gen out/Default --args='
  is_debug = false
  is_component_build = true
  symbol_level = 1
  enable_nacl = false
'

# 查看所有参数
gn args out/Debug --list

# 编辑参数 (打开编辑器)
gn args out/Debug
```

### 常用构建参数

```python
# args.gn 常用配置

# === 构建类型 ===
is_debug = true                    # Debug 构建
is_component_build = true          # 组件构建 (加快链接)

# === 优化 ===
is_official_build = false          # 官方构建
optimize_for_size = false          # 体积优化

# === 符号 ===
symbol_level = 2                   # 符号级别 (0-2)
use_debug_fission = true           # 分割调试信息

# === 平台 ===
target_os = "linux"                # linux, mac, win, android, ios
target_cpu = "x64"                 # x86, x64, arm, arm64

# === 功能开关 ===
enable_nacl = false                # Native Client
enable_remoting = false            # 远程桌面
use_aura = true                    # Aura 窗口系统

# === 开发选项 ===
enable_resource_allowlist_generation = false
treat_warnings_as_errors = false

# === macOS 特定 ===
mac_deployment_target = "10.13"
use_system_xcode = true

# === Windows 特定 ===
visual_studio_version = "2022"
windows_sdk_version = "10.0.22621.0"

# === Android 特定 ===
android32_ndk_api_level = 21
android64_ndk_api_level = 21

# === Electron 定制 ===
# 如果要编译 Electron
# import("//electron/build/args/all.gn")
```

### GN 命令

```bash
# 查看目标描述
gn desc out/Debug //chrome

# 查看目标依赖
gn deps out/Debug //chrome

# 查看目标引用
gn refs out/Debug //base

# 格式化 BUILD.gn
gn format BUILD.gn

# 检查构建文件
gn check out/Debug

# 清理构建
gn clean out/Debug
```

## ⚡ Ninja 编译

### 编译命令

```bash
# 编译 Chrome
ninja -C out/Debug chrome

# 编译所有
ninja -C out/Debug all

# 并行编译
ninja -C out/Debug -j$(nproc) chrome

# 使用 autoninja (自动选择并行度)
autoninja -C out/Debug chrome

# 编译特定目标
ninja -C out/Debug unit_tests
ninja -C out/Debug content_shell

# 查看编译命令
ninja -C out/Debug -t commands chrome

# 查看编译状态
ninja -C out/Debug -t query chrome

# 编译清理
ninja -C out/Debug -t clean

# 干运行 (查看将编译什么)
ninja -C out/Debug -n chrome
```

## 🐛 调试配置

### Debug 构建

```bash
# 生成 Debug 构建
gn gen out/Debug --args='
  is_debug = true
  symbol_level = 2
  enable_iterator_debugging = true
'

# 编译
ninja -C out/Debug chrome

# 运行
out/Debug/Chromium.app/Contents/MacOS/Chromium  # macOS
out/Debug/chrome                                 # Linux
out/Debug/chrome.exe                             # Windows
```

### 调试器配置

```bash
# LLDB (macOS/Linux)
lldb out/Debug/chrome
(lldb) run --disable-setuid-sandbox

# GDB (Linux)
gdb out/Debug/chrome
(gdb) run --disable-setuid-sandbox

# WinDbg (Windows)
windbg out/Debug/chrome.exe

# Visual Studio Debugger
devenv /debugexe out\Debug\chrome.exe
```

### 调试参数

```bash
# 常用调试参数

# 禁用沙箱 (调试需要)
--disable-setuid-sandbox
--no-sandbox

# 单进程模式
--single-process

# 禁用 GPU
--disable-gpu

# 启用日志
--enable-logging=stderr
--v=1

# 远程调试端口
--remote-debugging-port=9222

# 等待调试器
--wait-for-debugger
--wait-for-debugger-children

# 忽略证书错误
--ignore-certificate-errors

# 禁用扩展
--disable-extensions

# 指定用户数据目录
--user-data-dir=/tmp/chrome-debug
```

### 渲染进程调试

```bash
# 调试渲染进程

# 1. 启动 Chrome 并等待渲染进程
out/Debug/chrome --wait-for-debugger-children \
                 --renderer-startup-dialog

# 2. 渲染进程会弹出对话框显示 PID
# 3. 使用调试器附加到渲染进程
lldb -p <PID>

# 4. 在调试器中继续执行
(lldb) continue
```

## 📊 符号与崩溃分析

### 符号管理

```bash
# 生成符号文件
# Debug 构建已包含符号
# Release 构建可以单独生成符号

# Linux: 生成 breakpad 符号
# components/crash/content/tools/generate_breakpad_symbols.py \
#   --build-dir=out/Release \
#   --symbols-dir=out/Release/symbols \
#   --binary=out/Release/chrome

# macOS: dSYM
dsymutil out/Release/chrome -o out/Release/chrome.dSYM

# Windows: PDB
# 构建时自动生成
```

### 崩溃分析

```bash
# 查看崩溃报告
chrome://crashes

# Linux coredump
echo "/tmp/core.%e.%p" | sudo tee /proc/sys/kernel/core_pattern
ulimit -c unlimited
# 崩溃后
gdb out/Debug/chrome /tmp/core.chrome.12345

# macOS crash reports
# ~/Library/Logs/DiagnosticReports/

# Windows
# %LOCALAPPDATA%\Chromium\User Data\CrashPad\reports\

# 使用 minidump 分析
# breakpad/src/processor/minidump_stackwalk
```

### Address Sanitizer (ASan)

```bash
# 启用 ASan 检测内存错误
gn gen out/Asan --args='
  is_debug = false
  is_asan = true
'

ninja -C out/Asan chrome

# 运行时设置
export ASAN_OPTIONS="detect_leaks=1:detect_stack_use_after_return=1"
out/Asan/chrome
```
