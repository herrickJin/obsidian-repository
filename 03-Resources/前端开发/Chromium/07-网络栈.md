## 🌐 HTTP 请求流程

### 完整请求流程

```
┌─────────────────────────────────────────────────────────────┐
│                    HTTP 请求流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 请求初始化                                               │
│     └── 创建 URLRequest 对象                                │
│                                                             │
│  2. URL 解析                                                 │
│     └── 解析协议、主机、端口、路径                           │
│                                                             │
│  3. 缓存检查                                                 │
│     └── 检查是否有有效缓存                                   │
│                                                             │
│  4. DNS 解析                                                 │
│     └── 解析域名到 IP 地址                                   │
│                                                             │
│  5. 连接建立                                                 │
│     ├── 检查现有连接池                                       │
│     ├── 建立 TCP 连接                                        │
│     └── SSL/TLS 握手 (HTTPS)                                │
│                                                             │
│  6. 发送请求                                                 │
│     └── 发送 HTTP 请求头和请求体                             │
│                                                             │
│  7. 接收响应                                                 │
│     ├── 接收响应头                                           │
│     └── 接收响应体                                           │
│                                                             │
│  8. 响应处理                                                 │
│     ├── 缓存响应 (如适用)                                    │
│     └── 返回数据给请求者                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### URLRequest 生命周期

```cpp
// URLRequest 状态机

enum class URLRequestStatus {
  // 初始化
  IDLE,

  // 等待缓存检查
  WAITING_FOR_CACHE,

  // 等待连接
  WAITING_FOR_CONNECTION,

  // 发送请求
  SENDING_REQUEST,

  // 等待响应
  WAITING_FOR_RESPONSE,

  // 读取响应
  READING_RESPONSE,

  // 完成
  COMPLETED,

  // 错误
  FAILED
};

// 请求流程
URLRequest request(url);
request.set_delegate(delegate);
request.Start();  // 开始请求

// 回调
void OnResponseStarted(URLRequest* request, int net_error) {
  if (net_error == OK) {
    // 开始读取响应
    request->Read(buffer, buffer_size);
  }
}

void OnReadCompleted(URLRequest* request, int bytes_read) {
  if (bytes_read > 0) {
    // 处理数据
    ProcessData(buffer, bytes_read);
    // 继续读取
    request->Read(buffer, buffer_size);
  } else {
    // 读取完成
  }
}
```

## 📦 HTTP 协议栈

### HTTP/1.1

```
┌─────────────────────────────────────────────────────────────┐
│                    HTTP/1.1 特点                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特性:                                                       │
│  ├── 持久连接 (Keep-Alive)                                  │
│  ├── 管道化 (Pipelining) - 很少使用                         │
│  ├── 分块传输编码 (Chunked)                                 │
│  └── 压缩 (gzip, deflate)                                  │
│                                                             │
│  限制:                                                       │
│  ├── 队头阻塞 (Head-of-Line Blocking)                       │
│  ├── 每个域名连接数限制 (6-8 个)                            │
│  └── 冗余的请求头                                           │
│                                                             │
│  请求示例:                                                   │
│  GET /path HTTP/1.1                                        │
│  Host: example.com                                         │
│  User-Agent: Chrome/120.0                                  │
│  Accept: text/html                                         │
│  Connection: keep-alive                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### HTTP/2

```
┌─────────────────────────────────────────────────────────────┐
│                    HTTP/2 特点                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特性:                                                       │
│  ├── 多路复用 (Multiplexing)                                │
│  │   └── 单连接并行多个请求                                  │
│  ├── 二进制分帧 (Binary Framing)                            │
│  │   └── 更高效解析                                         │
│  ├── 头部压缩 (HPACK)                                       │
│  │   └── 减少重复头部                                       │
│  ├── 服务端推送 (Server Push)                               │
│  └── 请求优先级                                             │
│                                                             │
│  帧类型:                                                     │
│  ├── DATA: 数据帧                                           │
│  ├── HEADERS: 头部帧                                        │
│  ├── PRIORITY: 优先级帧                                     │
│  ├── RST_STREAM: 重置流                                     │
│  ├── SETTINGS: 设置帧                                       │
│  ├── PUSH_PROMISE: 推送承诺                                 │
│  ├── PING: 心跳                                             │
│  ├── GOAWAY: 关闭连接                                       │
│  └── WINDOW_UPDATE: 流量控制                                │
│                                                             │
│  流 (Stream):                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Connection (连接)                                    │   │
│  │  ├── Stream 1 (请求/响应)                           │   │
│  │  ├── Stream 3 (请求/响应)                           │   │
│  │  ├── Stream 5 (请求/响应)                           │   │
│  │  └── ...                                            │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### HTTP/3 (QUIC)

```
┌─────────────────────────────────────────────────────────────┐
│                   HTTP/3 (QUIC) 特点                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特性:                                                       │
│  ├── 基于 UDP 而非 TCP                                      │
│  ├── 0-RTT 连接建立                                         │
│  ├── 无队头阻塞                                             │
│  ├── 连接迁移 (网络切换不断连)                               │
│  └── 改进的拥塞控制                                         │
│                                                             │
│  协议栈对比:                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ HTTP/1.1, HTTP/2    │         HTTP/3               │   │
│  │  ─────────────      │       ────────────           │   │
│  │      HTTP           │           HTTP               │   │
│  │      TLS            │           QPACK              │   │
│  │      TCP            │          QUIC                │   │
│  │      IP             │      UDP    │  IP            │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  QUIC 优势:                                                  │
│  ├── 更快的连接建立                                          │
│  ├── 独立的流，互不阻塞                                      │
│  ├── 更好的移动端体验                                        │
│  └── 内置加密 (TLS 1.3)                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 缓存策略

```
┌─────────────────────────────────────────────────────────────┐
│                    缓存决策流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 检查缓存是否存在                                         │
│     │                                                       │
│     ├── 不存在 → 发起网络请求                                │
│     │                                                       │
│     └── 存在 → 检查新鲜度                                    │
│                                                             │
│  2. 检查新鲜度                                               │
│     │                                                       │
│     ├── 新鲜 (未过期) → 使用缓存                             │
│     │                                                       │
│     └── 过期 → 条件请求                                      │
│                                                             │
│  3. 条件请求                                                 │
│     │                                                       │
│     ├── If-Modified-Since (Last-Modified)                  │
│     │   └── 304 → 使用缓存                                  │
│     │   └── 200 → 更新缓存                                  │
│     │                                                       │
│     └── If-None-Match (ETag)                               │
│         └── 304 → 使用缓存                                  │
│         └── 200 → 更新缓存                                  │
│                                                             │
│  Cache-Control 指令:                                        │
│  ├── max-age=秒数: 最大新鲜时间                             │
│  ├── no-cache: 需要验证                                     │
│  ├── no-store: 不缓存                                       │
│  ├── public: 可被任何缓存存储                                │
│  ├── private: 只能被浏览器缓存                               │
│  └── immutable: 永不过期                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🍪 Cookie 管理

### Cookie 存储结构

```
┌─────────────────────────────────────────────────────────────┐
│                    Cookie 存储                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Cookie 属性:                                                │
│  ├── Name/Value: 名称和值                                   │
│  ├── Domain: 作用域名                                       │
│  ├── Path: 作用路径                                         │
│  ├── Expires/Max-Age: 过期时间                              │
│  ├── Secure: 仅 HTTPS                                       │
│  ├── HttpOnly: 不可 JavaScript 访问                         │
│  ├── SameSite: 跨站限制                                     │
│  │   ├── Strict: 完全禁止跨站发送                           │
│  │   ├── Lax: 允许顶级导航跨站发送                          │
│  │   └── None: 允许跨站发送 (需 Secure)                     │
│  └── Priority: 优先级 (low, medium, high)                   │
│                                                             │
│  存储:                                                       │
│  ├── SQLite 数据库 (Cookie 持久化)                          │
│  └── 内存映射 (快速访问)                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
