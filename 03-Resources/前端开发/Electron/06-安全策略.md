## 🔒 安全模型

### 为什么 Electron 需要特别关注安全？

```
┌─────────────────────────────────────────────────────────────┐
│                    Electron 安全挑战                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 访问系统资源                                             │
│     - 文件系统                                              │
│     - 网络请求                                              │
│     - 原生模块                                              │
│                                                             │
│  2. 加载远程内容                                             │
│     - XSS 攻击风险                                          │
│     - 远程代码执行                                          │
│                                                             │
│  3. 进程模型                                                 │
│     - 主进程拥有完整权限                                     │
│     - 渲染进程需要隔离                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## ⚠️ 危险配置 vs 安全配置

### 1. Context Isolation (上下文隔离)

```javascript
// ❌ 危险：禁用上下文隔离
new BrowserWindow({
  webPreferences: {
    contextIsolation: false
  }
})

// ✅ 安全：启用上下文隔离（默认）
new BrowserWindow({
  webPreferences: {
    contextIsolation: true  // 默认值
  }
})
```

**为什么重要？**

```
没有上下文隔离：
┌─────────────────────────────────────────┐
│ 渲染进程 JavaScript 环境                  │
│ ├── 网页脚本 (可能恶意)                   │
│ ├── Electron API                         │
│ └── preload 脚本                         │
│     ↑ 可以互相访问！                      │
└─────────────────────────────────────────┘

有上下文隔离：
┌─────────────────────────────────────────┐
│ 渲染进程 JavaScript 环境                  │
│ ├── 网页脚本 (隔离)                       │
│ └── preload 脚本 (隔离)                   │
│     ↑ 只能通过 contextBridge 通信         │
└─────────────────────────────────────────┘
```

### 2. Node Integration (Node 集成)

```javascript
// ❌ 危险：在渲染进程中启用 Node.js
new BrowserWindow({
  webPreferences: {
    nodeIntegration: true,
    contextIsolation: false
  }
})

// ✅ 安全：禁用 Node 集成（默认）
new BrowserWindow({
  webPreferences: {
    nodeIntegration: false,  // 默认值
    contextIsolation: true,
    preload: path.join(__dirname, 'preload.js')
  }
})
```

**为什么危险？**

```javascript
// 如果启用了 nodeIntegration，渲染进程可以直接执行：
const { exec } = require('child_process')
exec('rm -rf /')  // 灾难性的！

// 或者读取敏感文件：
const fs = require('fs')
const password = fs.readFileSync('/etc/passwd', 'utf-8')
```

### 3. enableRemoteModule

```javascript
// ❌ 危险：启用 remote 模块
new BrowserWindow({
  webPreferences: {
    enableRemoteModule: true
  }
})

// ✅ 安全：禁用 remote 模块
new BrowserWindow({
  webPreferences: {
    enableRemoteModule: false  // Electron 14+ 已移除
  }
})
```

> 注意：`remote` 模块在 Electron 14+ 已被移除。如果需要类似功能，使用 `@electron/remote` 包。

## 🛡️ 安全最佳实践

### 1. 安全的 BrowserWindow 配置

```javascript
const mainWindow = new BrowserWindow({
  webPreferences: {
    // ✅ 必须配置
    contextIsolation: true,    // 启用上下文隔离
    nodeIntegration: false,    // 禁用 Node 集成
    sandbox: true,             // 启用沙箱
    preload: path.join(__dirname, 'preload.js'),

    // ✅ 推荐配置
    webSecurity: true,         // 启用同源策略
    allowRunningInsecureContent: false,  // 禁止加载不安全内容

    // ⚠️ 根据需要配置
    javascript: true,          // 启用 JavaScript
    plugins: false,            // 禁用插件
    experimentalFeatures: false  // 禁用实验性功能
  }
})
```

### 2. 安全的 preload 脚本

```javascript
// preload.js
const { contextBridge, ipcRenderer } = require('electron')

// ✅ 使用白名单验证 channel
const validChannels = {
  send: ['toMain', 'save-file', 'open-dialog'],
  receive: ['fromMain', 'update-progress', 'file-changed'],
  invoke: ['get-user', 'read-file', 'write-file']
}

contextBridge.exposeInMainWorld('electronAPI', {
  // 安全的 send
  send: (channel, data) => {
    if (validChannels.send.includes(channel)) {
      ipcRenderer.send(channel, data)
    }
  },

  // 安全的 receive
  receive: (channel, callback) => {
    if (validChannels.receive.includes(channel)) {
      // 移除 event 对象，只传递数据
      ipcRenderer.on(channel, (event, ...args) => callback(...args))
    }
  },

  // 安全的 invoke
  invoke: (channel, data) => {
    if (validChannels.invoke.includes(channel)) {
      return ipcRenderer.invoke(channel, data)
    }
    return Promise.reject(new Error('Invalid channel'))
  }
})
```

### 3. 内容安全策略 (CSP)

```javascript
// 在主进程中设置 CSP
session.defaultSession.webRequest.onHeadersReceived((details, callback) => {
  callback({
    responseHeaders: {
      ...details.responseHeaders,
      'Content-Security-Policy': [
        "default-src 'self'",
        "script-src 'self'",
        "style-src 'self' 'unsafe-inline'",
        "img-src 'self' data: https:",
        "connect-src 'self' https://api.example.com",
        "font-src 'self' data:",
        "object-src 'none'",
        "base-uri 'self'",
        "form-action 'self'",
        "frame-ancestors 'none'"
      ].join('; ')
    }
  })
})

// 或在 HTML 中设置
// <meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self'">
```

### 4. 验证所有输入

```javascript
// main.js
const { dialog } = require('electron')
const path = require('path')

// ✅ 验证文件路径
ipcMain.handle('file:read', async (event, filePath) => {
  // 规范化路径
  const normalizedPath = path.normalize(filePath)

  // 检查是否在允许的目录内
  const allowedDir = app.getPath('userData')
  const resolvedPath = path.resolve(normalizedPath)

  if (!resolvedPath.startsWith(allowedDir)) {
    throw new Error('Access denied: path outside allowed directory')
  }

  // 检查文件是否存在
  if (!fs.existsSync(resolvedPath)) {
    throw new Error('File not found')
  }

  // 读取文件
  return fs.readFile(resolvedPath, 'utf-8')
})

// ✅ 验证 URL
ipcMain.handle('shell:openExternal', async (event, url) => {
  try {
    const parsedUrl = new URL(url)

    // 只允许特定协议
    if (!['http:', 'https:'].includes(parsedUrl.protocol)) {
      throw new Error('Invalid protocol')
    }

    // 可选：只允许特定域名
    const allowedDomains = ['example.com', 'api.example.com']
    if (!allowedDomains.some(d => parsedUrl.hostname.endsWith(d))) {
      throw new Error('Domain not allowed')
    }

    await shell.openExternal(url)
  } catch (error) {
    console.error('Invalid URL:', error)
  }
})
```

## 🚨 常见安全漏洞

### 1. XSS (跨站脚本攻击)

```javascript
// ❌ 危险：直接插入用户内容
element.innerHTML = userInput

// ✅ 安全：使用 textContent 或转义
element.textContent = userInput

// ✅ 或使用 DOMPurify 库
import DOMPurify from 'dompurify'
element.innerHTML = DOMPurify.sanitize(userInput)
```

### 2. 远程代码执行

```javascript
// ❌ 危险：加载远程内容并启用 Node
win.loadURL('https://example.com')
// 如果启用了 nodeIntegration，远程网站可以执行任意代码

// ✅ 安全：只加载本地内容
win.loadFile('index.html')

// ✅ 如果必须加载远程内容，使用 webview 标签并严格限制
// <webview src="https://example.com" nodeintegration="no" disablewebsecurity="yes"></webview>
```

### 3. 不安全的 webview

```javascript
// ❌ 危险：不安全的 webview 配置
<webview src="https://untrusted.com" nodeintegration></webview>

// ✅ 安全：限制 webview 权限
<webview
  src="https://untrusted.com"
  nodeintegration="no"
  disablewebsecurity="no"
  allowpopups="no"
></webview>

// 在主进程中限制
app.on('web-contents-created', (event, contents) => {
  contents.on('will-attach-webview', (event, webPreferences, params) => {
    // 禁用 Node 集成
    webPreferences.nodeIntegration = false
    webPreferences.contextIsolation = true

    // 验证 URL
    if (!params.src.startsWith('https://')) {
      event.preventDefault()
    }
  })
})
```

### 4. 导航劫持

```javascript
// 防止导航到恶意网站
mainWindow.webContents.on('will-navigate', (event, url) => {
  const parsedUrl = new URL(url)

  // 只允许导航到特定域名
  if (!parsedUrl.hostname.endsWith('myapp.com')) {
    event.preventDefault()
  }
})

// 防止新窗口打开
mainWindow.webContents.setWindowOpenHandler(({ url }) => {
  // 在默认浏览器中打开
  shell.openExternal(url)
  return { action: 'deny' }
})
```

## 🔐 安全清单

### BrowserWindow 配置

- [ ] `contextIsolation: true`
- [ ] `nodeIntegration: false`
- [ ] `sandbox: true`
- [ ] `webSecurity: true`
- [ ] `allowRunningInsecureContent: false`

### preload 脚本

- [ ] 使用 `contextBridge.exposeInMainWorld`
- [ ] 使用白名单验证 IPC channel
- [ ] 不暴露完整的 `ipcRenderer` 对象

### 内容安全

- [ ] 设置 CSP 头
- [ ] 验证所有用户输入
- [ ] 转义或清理 HTML 内容
- [ ] 验证文件路径和 URL

### 网络安全

- [ ] 使用 HTTPS
- [ ] 验证 SSL 证书
- [ ] 限制导航目标
- [ ] 限制新窗口打开

### 其他

- [ ] 禁用或移除 `remote` 模块
- [ ] 限制 `webview` 权限
- [ ] 定期更新 Electron 版本
- [ ] 代码签名和公证
