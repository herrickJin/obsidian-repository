# 编解码器

> 编解码器 (Codec) 负责音视频数据的压缩和解压缩，是 WebRTC 的核心组件

## 📌 核心概念

### 编码 (Encoding)
将原始音视频数据压缩为更小的格式，便于传输

### 解码 (Decoding)
将压缩的数据还原为可播放的音视频

---

## 🎤 音频编解码器

### Opus (推荐)

**WebRTC 默认音频编解码器**

| 特性 | 说明 |
|------|------|
| 采样率 | 8kHz - 48kHz |
| 比特率 | 6kbps - 510kbps |
| 帧大小 | 2.5ms - 60ms |
| 声道 | 单声道/立体声 |
| 延迟 | 5ms - 66.5ms |

```javascript
// SDP 中的 Opus 描述
a=rtpmap:111 opus/48000/2
a=fmtp:111 minptime=10;useinbandfec=1;usedtx=0

// fmtp 参数说明:
// minptime=10       - 最小打包时间 10ms
// useinbandfec=1    - 启用带内 FEC
// usedtx=0          - 禁用 DTX (不连续传输)
// stereo=1          - 启用立体声
// sprop-stereo=1    - 建议立体声
```

**Opus 特点**：
- ✅ 高质量低延迟
- ✅ 自适应比特率
- ✅ 支持 FEC 前向纠错
- ✅ 支持 DTX 静音检测
- ✅ 宽带/超宽带/全带

### 其他音频编解码器

| 编解码器 | 采样率 | 比特率 | 说明 |
|----------|--------|--------|------|
| PCMU (G.711 μ-law) | 8kHz | 64kbps | 传统电话，北美标准 |
| PCMA (G.711 A-law) | 8kHz | 64kbps | 传统电话，欧洲标准 |
| G.722 | 16kHz | 64kbps | 宽带音频 |
| iSAC | 16/32kHz | 10-32kbps | 宽带音频 |
| CN (Comfort Noise) | 8/16/32/48kHz | - | 舒适噪声 |

```javascript
// SDP 中的音频编解码器列表
m=audio 9 UDP/TLS/RTP/SAVPF 111 103 104 9 0 8 106 105 13 110

// 对应:
// 111 - Opus/48000/2
// 103 - iSAC/16000
// 104 - iSAC/32000
// 9   - G722/8000
// 0   - PCMU/8000
// 8   - PCMA/8000
// 106 - CN/32000
// 105 - CN/16000
// 13  - CN/8000
// 110 - telephone-event/48000 (DTMF)
```

## 🎬 视频编解码器

### VP8

**Google 开源的免费编解码器**

| 特性 | 说明 |
|------|------|
| 分辨率 | 最高 4K |
| 帧率 | 最高 60fps |
| 比特率 | 自适应 |
| 延迟 | 低 |

```javascript
// SDP 中的 VP8 描述
a=rtpmap:96 VP8/90000
a=rtcp-fb:96 goog-remb       // REMB 带宽估计
a=rtcp-fb:96 transport-cc    // 传输拥塞控制
a=rtcp-fb:96 ccm fir         // 完整帧请求
a=rtcp-fb:96 nack            // NACK 重传
a=rtcp-fb:96 nack pli        // 图像丢失指示
```

### VP9

**VP8 的升级版，更高效**

| 特性 | 说明 |
|------|------|
| 压缩效率 | 比 VP8 高 30-50% |
| 分辨率 | 最高 8K |
| 特性 | SVC 可分层编码 |

```javascript
// SDP 中的 VP9 描述
a=rtpmap:98 VP9/90000
a=rtcp-fb:98 goog-remb
a=rtcp-fb:98 transport-cc
a=rtcp-fb:98 ccm fir
a=rtcp-fb:98 nack
a=rtcp-fb:98 nack pli

// VP9 特有的 profile-id
a=fmtp:98 profile-id=0
```

### AV1

**最新一代免费编解码器**

| 特性 | 说明 |
|------|------|
| 压缩效率 | 比 VP9 高 20-30% |
| 分辨率 | 最高 8K |
| 开发者 | AOM (Apple, Google, Microsoft...) |

```javascript
// SDP 中的 AV1 描述
a=rtpmap:99 AV1/90000
a=rtcp-fb:99 goog-remb
a=rtcp-fb:99 transport-cc
a=rtcp-fb:99 ccm fir
a=rtcp-fb:99 nack
a=rtcp-fb:99 nack pli
```

### H.264 (AVC)

**最广泛使用的视频编解码器**

| 特性 | 说明 |
|------|------|
| 兼容性 | 所有浏览器和设备 |
| Profile | Baseline, Main, High |
| 分辨率 | 最高 4K |

```javascript
// SDP 中的 H.264 描述
a=rtpmap:100 H264/90000
a=fmtp:100 level-asymmetry-allowed=1;packetization-mode=1;profile-level-id=42e01f
a=rtcp-fb:100 goog-remb
a=rtcp-fb:100 transport-cc
a=rtcp-fb:100 ccm fir
a=rtcp-fb:100 nack
a=rtcp-fb:100 nack pli

// profile-level-id 说明:
// 42e01f = Constrained Baseline Profile, Level 3.1
// 42001f = Baseline Profile, Level 3.1
```

### H.265 (HEVC)

**H.264 的升级版**

| 特性 | 说明 |
|------|------|
| 压缩效率 | 比 H.264 高 50% |
| 兼容性 | Safari 支持 |
| 许可 | 需要专利授权 |

## 🔄 编解码器协商

### SDP 协商过程

```
1. Offer 端列出支持的所有编解码器（按优先级排序）
2. Answer 端选择支持的编解码器（保持优先级）
3. 双方使用第一个共同支持的编解码器
```

### 设置编解码器偏好

```javascript
// 方式1: 使用 setCodecPreferences (推荐)
async function setPreferredCodec(pc, kind, mimeType) {
  const transceivers = pc.getTransceivers();
  const transceiver = transceivers.find(t => t.receiver.track.kind === kind);

  if (transceiver) {
    const capabilities = RTCRtpReceiver.getCapabilities(kind);
    const codecs = capabilities.codecs;

    // 过滤并排序编解码器
    const preferredCodecs = codecs.filter(c => c.mimeType === mimeType);
    const otherCodecs = codecs.filter(c => c.mimeType !== mimeType);

    transceiver.setCodecPreferences([...preferredCodecs, ...otherCodecs]);
  }
}

// 使用
const pc = new RTCPeerConnection();
// 添加轨道...

// 设置 VP9 为首选
await setPreferredCodec(pc, 'video', 'video/VP9');

// 然后创建 offer/answer
```

### 查看协商结果

```javascript
// 查看实际使用的编解码器
async function getNegotiatedCodec(pc) {
  const transceivers = pc.getTransceivers();

  for (const transceiver of transceivers) {
    const stats = await pc.getStats();
    stats.forEach(report => {
      if (report.type === 'inbound-rtp' || report.type === 'outbound-rtp') {
        console.log(`${report.kind} 编解码器:`, report.codecId);

        // 查找编解码器详情
        stats.forEach(codec => {
          if (codec.id === report.codecId) {
            console.log('编解码器详情:', codec.mimeType, codec.clockRate);
          }
        });
      }
    });
  }
}
```

## 🎛️ 编码参数控制

### 比特率控制

```javascript
// 设置最大比特率
async function setMaxBitrate(pc, kind, maxBitrate) {
  const senders = pc.getSenders();
  const sender = senders.find(s => s.track?.kind === kind);

  if (sender) {
    const parameters = sender.getParameters();

    if (!parameters.encodings) {
      parameters.encodings = [{}];
    }

    parameters.encodings[0].maxBitrate = maxBitrate;
    await sender.setParameters(parameters);
  }
}

// 使用: 设置视频最大比特率为 2Mbps
await setMaxBitrate(pc, 'video', 2000000);
```

### 分辨率和帧率

```javascript
// 使用 getUserMedia 限制
const stream = await navigator.mediaDevices.getUserMedia({
  video: {
    width: { ideal: 1280 },
    height: { ideal: 720 },
    frameRate: { ideal: 30, max: 30 }
  }
});

// 动态调整
const videoTrack = stream.getVideoTracks()[0];
await videoTrack.applyConstraints({
  width: 640,
  height: 360,
  frameRate: 15
});
```

### Simulcast (多层编码)

```javascript
// 启用 Simulcast（发送多个分辨率）
const pc = new RTCPeerConnection();

const transceiver = pc.addTransceiver(videoTrack, {
  direction: 'sendrecv',
  sendEncodings: [
    { rid: 'h', maxBitrate: 900000 },   // 高清
    { rid: 'm', maxBitrate: 300000, scaleResolutionDownBy: 2 },  // 中等
    { rid: 'l', maxBitrate: 100000, scaleResolutionDownBy: 4 }   // 低
  ]
});
```

## 📦 RTX 重传机制

```javascript
// RTX (Retransmission) 用于 NACK 重传
a=rtpmap:97 rtx/90000
a=fmtp:97 apt=96  // apt 指向关联的 VP8 payload type

// 工作流程:
// 1. 接收端发现丢包，发送 NACK
// 2. 发送端通过 RTX 重新发送丢失的包
// 3. 接收端重组完整帧
```

## 🔧 音频处理功能

### 回声消除 (AEC)

```javascript
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    echoCancellation: true,  // 启用回声消除
    echoCancellationType: 'browser'  // 'browser' | 'system'
  }
});
```

### 噪声抑制 (NS)

```javascript
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    noiseSuppression: true  // 启用噪声抑制
  }
});
```

### 自动增益控制 (AGC)

```javascript
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    autoGainControl: true  // 启用自动增益控制
  }
});
```
