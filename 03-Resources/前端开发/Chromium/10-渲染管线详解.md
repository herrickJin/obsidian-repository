## 🎨 渲染树 (Render Tree)

### 渲染树构建

```
DOM + CSSOM → Render Tree

特点:
├── 只包含可见元素
├── 排除 display: none 的元素
├── 包含 visibility: hidden 的元素
└── 合并 DOM 结构和计算样式

示例:
┌─────────────────────────────────────────┐
│ RenderObject (div#app)                  │
│ ├── RenderObject (h1)                   │
│ │   └── RenderText ("Title")            │
│ └── RenderObject (p)                    │
│     └── RenderText ("Content")          │
└─────────────────────────────────────────┘

不包含:
├── <head> 及其内容
├── display: none 的元素
└── <script> 标签
```

### 布局计算

```
┌─────────────────────────────────────────────────────────────┐
│                      布局过程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入: Render Tree + 计算样式                               │
│  输出: Layout Tree (包含位置和大小)                          │
│                                                             │
│  计算内容:                                                   │
│  ├── 元素位置 (x, y)                                        │
│  ├── 元素大小 (width, height)                               │
│  ├── 内边距 (padding)                                       │
│  ├── 边框 (border)                                          │
│  ├── 外边距 (margin)                                        │
│  └── 文本行高、字间距等                                      │
│                                                             │
│  布局算法:                                                   │
│  ├── Block Layout (块布局)                                  │
│  ├── Inline Layout (行内布局)                               │
│  ├── Flex Layout (弹性布局)                                 │
│  ├── Grid Layout (网格布局)                                 │
│  └── Table Layout (表格布局)                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 触发重排的操作

```javascript
// 触发重排 (Layout/Reflow) 的操作

// 1. 改变元素尺寸
element.style.width = '100px'
element.style.height = '100px'
element.style.padding = '10px'
element.style.margin = '10px'

// 2. 改变元素位置
element.style.top = '10px'
element.style.left = '10px'

// 3. 添加/删除元素
parent.appendChild(child)
parent.removeChild(child)

// 4. 改变字体
element.style.fontSize = '16px'
element.style.fontFamily = 'Arial'

// 5. 改变窗口大小
window.addEventListener('resize', handler)

// 6. 读取布局属性 (强制同步布局)
const width = element.offsetWidth
const height = element.offsetHeight
const rect = element.getBoundingClientRect()
const scroll = element.scrollTop

// 7. 改变类名 (可能影响样式)
element.className = 'new-class'

// 8. 内容变化
element.textContent = 'new text'
element.innerHTML = '<span>new</span>'
```

### 布局抖动 (Layout Thrashing)

```javascript
// ❌ 布局抖动: 读写交替导致多次重排

for (let i = 0; i < elements.length; i++) {
  elements[i].style.width = elements[i].offsetWidth + 10 + 'px'
  // 每次循环都触发:
  // 1. 读取 offsetWidth (强制布局)
  // 2. 设置 width (标记脏)
  // 3. 下次读取又强制布局
}

// ✅ 批量读取，批量写入

// 先批量读取
const widths = []
for (let i = 0; i < elements.length; i++) {
  widths[i] = elements[i].offsetWidth
}

// 再批量写入
for (let i = 0; i < elements.length; i++) {
  elements[i].style.width = widths[i] + 10 + 'px'
}

// ✅ 使用 requestAnimationFrame
function updateWidths() {
  const widths = elements.map(el => el.offsetWidth)
  requestAnimationFrame(() => {
    elements.forEach((el, i) => {
      el.style.width = widths[i] + 10 + 'px'
    })
  })
}
```

## 🖌️ 绘制 (Paint)

### 绘制过程

```
┌─────────────────────────────────────────────────────────────┐
│                      绘制过程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入: Layout Tree                                          │
│  输出: Paint Record (绘制指令列表)                           │
│                                                             │
│  绘制顺序 (从后到前):                                        │
│  1. 背景颜色                                                │
│  2. 背景图片                                                │
│  3. 边框                                                    │
│  4. 子元素                                                  │
│  5. 轮廓 (outline)                                          │
│  6. 文本和装饰                                              │
│                                                             │
│  绘制指令示例:                                               │
│  ├── drawRect(x, y, w, h, color)                           │
│  ├── drawText(text, x, y, style)                           │
│  ├── drawImage(image, x, y, w, h)                          │
│  └── clip(path)                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 触发重绘的操作

```javascript
// 触发重绘 (Repaint) 但不重排的操作

// 颜色变化
element.style.color = 'red'
element.style.backgroundColor = 'blue'
element.style.borderColor = 'green'

// 背景图片
element.style.backgroundImage = 'url(...)'

// 可见性
element.style.visibility = 'hidden'  // 不影响布局
element.style.opacity = '0.5'

// 轮廓
element.style.outline = '1px solid red'

// 阴影
element.style.boxShadow = '5px 5px 10px black'

// 文本装饰
element.style.textDecoration = 'underline'
```

## 🔀 合成 (Composite)

### 图层合成

```
┌─────────────────────────────────────────────────────────────┐
│                      合成过程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入: Paint Record                                         │
│  输出: 合成后的图像                                          │
│                                                             │
│  步骤:                                                       │
│  1. 图层树构建                                               │
│     └── 将页面分成多个图层                                   │
│                                                             │
│  2. 图层绘制                                                 │
│     └── 每个图层独立绘制                                     │
│                                                             │
│  3. 图层合成                                                 │
│     └── 将所有图层按顺序合成                                 │
│                                                             │
│  优势:                                                       │
│  ├── 只更新变化的图层                                        │
│  ├── GPU 加速                                               │
│  ├── 滚动优化                                               │
│  └── 动画性能                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 合成层创建条件

```css
/* 创建合成层的 CSS 属性 */

/* 3D 变换 */
.layer-3d {
  transform: translateZ(0);
  transform: translate3d(0, 0, 0);
  transform: rotateX(45deg);
}

/* will-change */
.will-change-layer {
  will-change: transform;
  will-change: opacity;
}

/* CSS 动画 (transform/opacity) */
@keyframes slide {
  from { transform: translateX(0); }
  to { transform: translateX(100px); }
}
.animated {
  animation: slide 1s;
}

/* position: fixed */
.fixed-layer {
  position: fixed;
}

/* video, canvas, iframe */
video, canvas, iframe {
  /* 自动创建合成层 */
}

/* backface-visibility: hidden */
.backface {
  backface-visibility: hidden;
}

/* opacity 动画 */
@keyframes fade {
  from { opacity: 0; }
  to { opacity: 1; }
}
.fade-animation {
  animation: fade 1s;
}
```

### 合成层陷阱

```css
/* ❌ 过度创建合成层 */

/* 每个元素都创建合成层会消耗大量内存 */
.too-many-layers div {
  will-change: transform;
}

/* ✅ 只在需要时使用 */

/* 动画结束后移除 will-change */
.animated {
  will-change: transform;
}
.animated:hover {
  transform: scale(1.1);
}
.animated:not(:hover) {
  will-change: auto;  /* 移除 */
}
```

## ⚡ 光栅化 (Rasterization)

### 光栅化过程

```
┌─────────────────────────────────────────────────────────────┐
│                     光栅化过程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  矢量图形 → 位图 (像素)                                      │
│                                                             │
│  方式:                                                       │
│  ├── CPU 光栅化                                             │
│  │   └── 使用 Skia CPU 后端                                 │
│  │   └── 适合简单图形                                        │
│  │                                                          │
│  └── GPU 光栅化                                             │
│      └── 使用 GPU 加速                                       │
│      └── 适合复杂图形                                        │
│                                                             │
│  分块光栅化 (Tile Rasterization):                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                    页面                              │   │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐              │   │
│  │  │ Tile │ │ Tile │ │ Tile │ │ Tile │              │   │
│  │  │  0   │ │  1   │ │  2   │ │  3   │              │   │
│  │  └──────┘ └──────┘ └──────┘ └──────┘              │   │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐              │   │
│  │  │ Tile │ │ Tile │ │ Tile │ │ Tile │              │   │
│  │  │  4   │ │  5   │ │  6   │ │  7   │              │   │
│  │  └──────┘ └──────┘ └──────┘ └──────┘              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  优势:                                                       │
│  ├── 只光栅化可见区域                                        │
│  ├── 按需光栅化                                              │
│  └── 支持渐进式渲染                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 性能优化代码示例

```javascript
// ✅ 使用 requestAnimationFrame 进行动画
function animate() {
  // 读取布局信息
  const width = element.offsetWidth

  requestAnimationFrame(() => {
    // 写入样式
    element.style.transform = `translateX(${width}px)`
  })
}

// ✅ 使用 transform 代替 top/left
// ❌
element.style.left = '100px'
element.style.top = '100px'

// ✅
element.style.transform = 'translate(100px, 100px)'

// ✅ 使用 classList 批量修改
// ❌
element.style.width = '100px'
element.style.height = '100px'
element.style.background = 'red'

// ✅
element.classList.add('active')

// ✅ 使用 DocumentFragment 批量添加
// ❌
for (let i = 0; i < 1000; i++) {
  const div = document.createElement('div')
  container.appendChild(div)  // 每次都触发重排
}

// ✅
const fragment = document.createDocumentFragment()
for (let i = 0; i < 1000; i++) {
  const div = document.createElement('div')
  fragment.appendChild(div)
}
container.appendChild(fragment)  // 只触发一次重排

// ✅ 使用 CSS Containment
.isolated {
  contain: layout;  /* 或 strict, content */
}
```

## 📊 性能分析

### Chrome DevTools Performance

```
使用 DevTools 分析渲染性能:

1. 打开 Performance 面板
2. 点击 Record
3. 执行操作
4. 停止录制
5. 分析结果

关注指标:
├── FPS (帧率)
│   └── 目标: 60 FPS (16.67ms/帧)
│
├── Main Thread
│   ├── Script 执行时间
│   ├── Layout 时间
│   ├── Paint 时间
│   └── Composite 时间
│
├── Layout Shift (布局偏移)
│   └── 目标: CLS < 0.1
│
└── Long Tasks
    └── 超过 50ms 的任务
```

### 渲染分析 API

```javascript
// Performance API
const paintEntries = performance.getEntriesByType('paint')
console.log('First Paint:', paintEntries.find(e => e.name === 'first-paint'))
console.log('First Contentful Paint:', paintEntries.find(e => e.name === 'first-contentful-paint'))

// Layout Instability API
let clsScore = 0
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (!entry.hadRecentInput) {
      clsScore += entry.value
    }
  }
  console.log('CLS:', clsScore)
})
observer.observe({ type: 'layout-shift', buffered: true })

// Long Tasks
const longTaskObserver = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log('Long Task:', entry.duration, 'ms')
  }
})
longTaskObserver.observe({ type: 'longtask', buffered: true })
```
