# Electron 架构理解

## 🎯 学习目标

- 理解 Electron 的核心组成
- 掌握主进程和渲染进程的区别
- 了解进程模型和通信机制

---

## 📖 核心概念

### Electron 是什么？

Electron = **Chromium** + **Node.js** + **Native APIs**

```
┌─────────────────────────────────────────────────────┐
│                    Electron 应用                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────┐  ┌─────────────┐  ┌────────────┐  │
│  │ Chromium    │  │ Node.js     │  │ Native API │  │
│  │ 渲染引擎    │  │ 后端运行时  │  │ 系统集成   │  │
│  └─────────────┘  └─────────────┘  └────────────┘  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 主进程 vs 渲染进程

| 特性 | 主进程 (Main Process) | 渲染进程 (Renderer Process) |
|------|----------------------|---------------------------|
| 数量 | 只有 1 个 | 可以有多个 |
| 入口文件 | main.js / index.js | HTML 文件 |
| Node.js | 完整支持 | 默认禁用（安全考虑） |
| 作用 | 创建窗口、系统交互 | 渲染页面、用户交互 |
| API | 所有 Electron API | 渲染相关 API |
| 通信 | ipcMain | ipcRenderer |

## 🔧 基本结构

### 最小化 Electron 应用

```
my-electron-app/
├── package.json        # 项目配置
├── main.js            # 主进程入口
├── preload.js         # 预加载脚本
└── index.html         # 渲染进程页面
```

### package.json

```json
{
  "name": "my-electron-app",
  "version": "1.0.0",
  "main": "main.js",
  "scripts": {
    "start": "electron ."
  },
  "devDependencies": {
    "electron": "^28.0.0"
  }
}
```

### main.js (主进程)

```javascript
const { app, BrowserWindow } = require('electron')
const path = require('path')

// 创建窗口
function createWindow() {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,  // 安全：上下文隔离
      nodeIntegration: false   // 安全：禁用 Node 集成
    }
  })

  win.loadFile('index.html')
}

// 应用准备就绪
app.whenReady().then(() => {
  createWindow()

  // macOS 特殊处理
  app.on('activate', () => {
    if (BrowserWindow.getAllWindows().length === 0) {
      createWindow()
    }
  })
})

// 关闭所有窗口时退出（Windows & Linux）
app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit()
  }
})
```

### preload.js (预加载脚本)

```javascript
const { contextBridge, ipcRenderer } = require('electron')

// 安全地暴露 API 给渲染进程
contextBridge.exposeInMainWorld('electronAPI', {
  // 示例：发送消息到主进程
  send: (channel, data) => ipcRenderer.send(channel, data),
  // 示例：接收主进程消息
  receive: (channel, callback) => {
    ipcRenderer.on(channel, (event, ...args) => callback(...args))
  }
})
```

### index.html (渲染进程)

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Hello Electron</title>
</head>
<body>
  <h1>Hello Electron!</h1>
  <button id="btn">点击发送消息</button>

  <script>
    // 使用 preload 暴露的 API
    document.getElementById('btn').addEventListener('click', () => {
      window.electronAPI.send('greeting', 'Hello from renderer!')
    })
  </script>
</body>
</html>
```

## 🔑 关键要点

### 1. 进程隔离的重要性

```
为什么需要进程隔离？

1. 安全性：渲染进程处理不可信的网页内容
2. 稳定性：一个渲染进程崩溃不影响其他进程
3. 性能：多进程利用多核 CPU

Chromium 的多进程架构：
- 每个标签页/窗口 = 独立进程
- GPU 进程独立
- 插件进程独立
```

### 2. Chromium 版本对应

| Electron | Chromium | Node.js | V8 |
|----------|----------|---------|-----|
| 28.x | 120 | 18.17 | 12.0 |
| 27.x | 118 | 18.17 | 11.8 |
| 26.x | 116 | 18.16 | 11.2 |

> 查看: https://www.electronjs.org/docs/latest/tutorial/electron-timelines

### 3. 生命周期事件

```javascript
// 应用生命周期
app.on('will-finish-launching')  // 即将完成启动
app.on('ready')                   // 应用准备就绪
app.on('window-all-closed')       // 所有窗口关闭
app.on('before-quit')             // 即将退出
app.on('will-quit')               // 将要退出
app.on('quit')                    // 退出完成

// 窗口生命周期
win.on('close')      // 窗口即将关闭
win.on('closed')     // 窗口已关闭
win.on('ready-to-show') // 窗口准备好显示
```
