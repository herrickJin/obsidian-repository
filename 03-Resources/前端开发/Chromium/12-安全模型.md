## 📦 沙箱机制 (Sandbox)

### 沙箱目标

```
┌─────────────────────────────────────────────────────────────┐
│                     沙箱目标                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  核心原则:                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │         不信任渲染进程的任何输入                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  限制范围:                                                   │
│  ├── 文件系统访问                                            │
│  │   └── 只能访问特定目录/文件                              │
│  │                                                         │
│  ├── 网络访问                                                │
│  │   └── 必须通过 Browser 进程                              │
│  │                                                         │
│  ├── 系统调用                                                │
│  │   └── 只允许安全的系统调用                               │
│  │                                                         │
│  ├── 进程间通信                                              │
│  │   └── 只能通过 Mojo/IPC                                  │
│  │                                                         │
│  └── 资源使用                                                │
│      └── CPU, 内存限制                                       │
│                                                             │
│  攻击假设:                                                   │
│  └── 即使渲染进程被完全控制，也无法危害系统                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Linux seccomp 示例

```cpp
// Linux seccomp-bpf 过滤规则

// 允许的系统调用
switch (sysno) {
  // 基本操作
  case __NR_read:
  case __NR_write:
  case __NR_close:
  case __NR_exit:
  case __NR_exit_group:
    return Allow();

  // 内存操作
  case __NR_mmap:
  case __NR_munmap:
  case __NR_mprotect:
    return Allow();

  // 线程操作
  case __NR_clone:
  case __NR_futex:
    return Allow();

  // 时间
  case __NR_clock_gettime:
  case __NR_gettimeofday:
    return Allow();

  // 禁止的危险调用
  case __NR_open:
  case __NR_openat:
  case __NR_execve:
  case __NR_socket:
    return Block();

  default:
    // 其他调用需要审查
    return Restrict();
}
```

### macOS Seatbelt

```
// macOS 沙箱配置文件示例

(version 1)
(deny default)

;; 允许的基本操作
(allow process-exec
  (literal "/System/Library/Frameworks/...")
  (subpath "/Applications/Chrome.app"))

;; 文件系统限制
(allow file-read-data
  (literal "/etc/hosts")
  (literal "/etc/resolv.conf"))

(allow file-write-data
  (subpath "/Users/.../Library/Application Support/Google/Chrome"))

;; 网络限制
(allow network-outbound
  (literal "/private/var/run/mDNSResponder")
  (remote tcp))

;; 进程间通信
(allow ipc-posix-shm)
(allow mach-lookup
  (global-name "com.apple.system.opendirectoryd.api"))
```

## 🌐 Site Isolation (站点隔离)

### 为什么需要站点隔离？

```
┌─────────────────────────────────────────────────────────────┐
│                  Spectre 攻击                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  攻击原理:                                                   │
│  1. 恶意网站执行 JavaScript                                  │
│  2. 利用 CPU 推测执行                                        │
│  3. 绕过沙箱读取进程内存                                      │
│  4. 获取其他网站的数据                                       │
│                                                             │
│  传统进程模型:                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Renderer Process                       │   │
│  │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐  │   │
│  │  │ example.com │ │ evil.com    │ │ bank.com    │  │   │
│  │  │             │ │ (恶意)      │ │ (敏感数据)   │  │   │
│  │  └─────────────┘ └─────────────┘ └─────────────┘  │   │
│  │              共享同一进程内存                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  站点隔离后:                                                 │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ Renderer #1 │ │ Renderer #2 │ │ Renderer #3 │          │
│  │ example.com │ │ evil.com    │ │ bank.com    │          │
│  └─────────────┘ └─────────────┘ └─────────────┘          │
│           独立进程，内存隔离                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 站点隔离策略

```
┌─────────────────────────────────────────────────────────────┐
│                  站点隔离策略                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  隔离级别:                                                   │
│  ├── Site (站点): eTLD+1                                   │
│  │   └── example.com, sub.example.com 共享                 │
│  │                                                         │
│  └── Origin (源): 协议 + 域名 + 端口                        │
│      └── https://a.example.com ≠ https://b.example.com    │
│                                                             │
│  隔离范围:                                                   │
│  ├── 主框架                                                 │
│  ├── 跨源 iframe                                            │
│  ├── 跨源弹窗                                               │
│  └── Service Worker                                         │
│                                                             │
│  特殊处理:                                                   │
│  ├── about:blank - 继承父框架                               │
│  ├── data: URL - 独立进程                                   │
│  └── 扩展页面 - 独立进程                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

│  │  └──────────────────┘  │  │  └──────────────────┘  │   │

│           通过 IPC 通信                                      │
│                                                             │

## 🔐 同源策略 (SOP)

### 同源定义

```
┌─────────────────────────────────────────────────────────────┐
│                    同源定义                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  源 (Origin) = 协议 + 域名 + 端口                           │
│                                                             │
│  示例对比:                                                   │
│  当前页面: https://www.example.com/page                    │
│                                                             │
│  URL                                    同源?               │
│  ─────────────────────────────────────────────────────      │
│  https://www.example.com/other          ✅ 同源             │
│  https://example.com/page               ❌ 不同子域名       │
│  https://www.example.com:443/page       ✅ 同源 (默认端口)  │
│  http://www.example.com/page            ❌ 不同协议         │
│  https://www.example.com:8080/page      ❌ 不同端口         │
│                                                             │
│  特殊情况:                                                   │
│  ├── IE: 端口不影响同源判断                                  │
│  └── 子域名: 可通过 document.domain 修改 (已废弃)           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### SOP 限制

```
┌─────────────────────────────────────────────────────────────┐
│                   同源策略限制                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  DOM 访问:                                                   │
│  ├── 同源: 可读写                                           │
│  └── 跨源: 不可访问                                         │
│                                                             │
│  Cookie:                                                     │
│  ├── 同源: 可访问                                           │
│  └── 跨源: 受 Domain, Path, SameSite 限制                   │
│                                                             │
│  Web Storage:                                                │
│  ├── LocalStorage: 源隔离                                   │
│  └── SessionStorage: 源 + 标签页隔离                        │
│                                                             │
│  XMLHttpRequest:                                             │
│  ├── 同源: 可发送                                           │
│  └── 跨源: 需要 CORS                                        │
│                                                             │
│  Fetch:                                                      │
│  ├── 同源: 可发送                                           │
│  └── 跨源: 需要 CORS                                        │
│                                                             │
│  字体:                                                       │
│  └── 跨源字体需要 CORS                                      │
│                                                             │
│  WebGL 贴图:                                                 │
│  └── 跨源图片需要 CORS                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### CORS 流程

```
┌─────────────────────────────────────────────────────────────┐
│                     CORS 流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  简单请求 (Simple Request):                                  │
│                                                             │
│  Browser                              Server                │
│    │                                    │                   │
│    │  GET /api HTTP/1.1                 │                   │
│    │  Origin: https://example.com       │                   │
│    │ ─────────────────────────────────> │                   │
│    │                                    │                   │
│    │  HTTP/1.1 200 OK                   │                   │
│    │  Access-Control-Allow-Origin: *    │                   │
│    │ <───────────────────────────────── │                   │
│                                                             │
│  预检请求 (Preflight):                                       │
│                                                             │
│  Browser                              Server                │
│    │                                    │                   │
│    │  OPTIONS /api HTTP/1.1             │                   │
│    │  Origin: https://example.com       │                   │
│    │  Access-Control-Request-Method: POST                  │
│    │  Access-Control-Request-Headers: X-Custom             │
│    │ ─────────────────────────────────> │                   │
│    │                                    │                   │
│    │  HTTP/1.1 200 OK                   │                   │
│    │  Access-Control-Allow-Origin: *    │                   │
│    │  Access-Control-Allow-Methods: POST                   │
│    │  Access-Control-Allow-Headers: X-Custom               │
│    │ <───────────────────────────────── │                   │
│    │                                    │                   │
│    │  POST /api HTTP/1.1                │                   │
│    │ ─────────────────────────────────> │                   │
│    │                                    │                   │
│    │  Response                          │                   │
│    │ <───────────────────────────────── │                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### CORS Headers

```
┌─────────────────────────────────────────────────────────────┐
│                    CORS Headers                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  请求头:                                                     │
│  ├── Origin: 请求来源                                       │
│  ├── Access-Control-Request-Method: 预检请求方法            │
│  └── Access-Control-Request-Headers: 预检请求头             │
│                                                             │
│  响应头:                                                     │
│  ├── Access-Control-Allow-Origin: 允许的源                  │

│  │                                                         │
│  ├── Access-Control-Allow-Methods: 允许的方法               │
│  │   └── GET, POST, PUT, DELETE                            │
│  │                                                         │
│  ├── Access-Control-Allow-Headers: 允许的请求头             │
│  │   └── Content-Type, X-Custom-Header                     │
│  │                                                         │
│  ├── Access-Control-Expose-Headers: 暴露给 JS 的响应头      │
│  │   └── X-Custom-Response-Header                          │
│  │                                                         │
│  ├── Access-Control-Allow-Credentials: 允许携带凭证         │
│  │   └── true                                              │
│  │                                                         │
│  └── Access-Control-Max-Age: 预检缓存时间                   │
│      └── 86400 (秒)                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🍪 安全 Cookie

### Cookie 安全属性

```
┌─────────────────────────────────────────────────────────────┐
│                   Cookie 安全属性                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Secure:                                                     │
│  ├── 只通过 HTTPS 发送                                      │
│  └── 防止中间人攻击                                         │
│                                                             │
│  HttpOnly:                                                   │
│  ├── JavaScript 无法访问                                    │
│  └── 防止 XSS 窃取 Cookie                                   │
│                                                             │
│  SameSite:                                                   │
│  ├── Strict: 完全禁止跨站发送                               │
│  │   └── 最安全，但影响用户体验                             │
│  │                                                         │
│  ├── Lax: 允许顶级导航跨站发送 (默认)                       │
│  │   └── 平衡安全与体验                                     │
│  │                                                         │
│  └── None: 允许跨站发送                                     │
│      └── 需要 Secure                                        │
│                                                             │
│  推荐配置:                                                   │
│  Set-Cookie: session=abc123;                                │
│              Secure;                                        │
│              HttpOnly;                                      │
│              SameSite=Lax;                                  │
│              Path=/;                                        │
│              Domain=example.com                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🔧 安全配置

### Content Security Policy (CSP)

```html
<!-- CSP 配置示例 -->

<!-- 通过 meta 标签 -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self';
               script-src 'self' https://cdn.example.com;
               style-src 'self' 'unsafe-inline';
               img-src 'self' data: https:;
               connect-src 'self' https://api.example.com;
               font-src 'self';
               object-src 'none';
               frame-ancestors 'none';
               base-uri 'self';
               form-action 'self';">

<!-- 通过 HTTP 响应头 -->
Content-Security-Policy:
  default-src 'self';
  script-src 'self' 'nonce-abc123';
  style-src 'self';
  img-src 'self' data:;
  connect-src 'self' https://api.example.com;
  font-src 'self';
  object-src 'none';
  frame-ancestors 'none';
  report-uri /csp-report;
```
