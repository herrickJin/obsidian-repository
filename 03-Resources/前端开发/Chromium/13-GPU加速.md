## 🏗️ GPU 进程架构

### 为什么需要独立 GPU 进程？

```
┌─────────────────────────────────────────────────────────────┐
│                   GPU 进程设计理由                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  稳定性:                                                     │
│  ├── GPU 驱动程序可能不稳定                                  │
│  ├── GPU 崩溃不影响浏览器主进程                              │
│  └── 可以恢复 GPU 进程                                       │
│                                                             │
│  安全性:                                                     │
│  ├── GPU 进程权限受限                                        │
│  ├── 限制对系统的访问                                        │
│  └── 沙箱隔离                                                │
│                                                             │
│  多进程共享:                                                 │
│  ├── 多个渲染进程共享 GPU                                    │
│  ├── 统一资源管理                                            │
│  └── 避免资源竞争                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 命令类型

```cpp
// GPU 命令类型

// 清屏
cmd::Clear clear_cmd;
clear_cmd.SetMask(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

// 绘制
cmd::DrawArrays draw_cmd;
draw_cmd.SetMode(GL_TRIANGLES);
draw_cmd.SetFirst(0);
draw_cmd.SetCount(3);

// 设置状态
cmd::Enable enable_cmd;
enable_cmd.SetCapability(GL_BLEND);

// 纹理操作
cmd::BindTexture bind_texture;
bind_texture.SetTarget(GL_TEXTURE_2D);
bind_texture.SetTexture(texture_id);

// 统一缓冲区
cmd::Uniform1f uniform_cmd;
uniform_cmd.SetLocation(location);
uniform_cmd.SetX(value);
```

## ⚡ GPU 光栅化

### 光栅化方式对比

```
┌─────────────────────────────────────────────────────────────┐
│                   光栅化方式对比                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  CPU 光栅化:                                                 │
│  ├── 在 Raster 线程执行                                     │
│  ├── 使用 Skia CPU 后端                                     │
│  ├── 适合简单内容                                            │
│  └── 内存效率高                                              │
│                                                             │
│  GPU 光栅化:                                                 │
│  ├── 在 GPU 执行                                            │
│  ├── 使用 Skia GPU 后端                                     │
│  ├── 适合复杂内容                                            │
│  ├── 支持并发光栅化                                          │
│  └── 更好的动画性能                                          │
│                                                             │
│  OOP (Out-of-Process) 光栅化:                               │
│  ├── 在 GPU 进程执行                                        │
│  ├── 更好的稳定性                                            │
│  └── 内存共享优化                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### GPU 命令行开关

```javascript
// Electron 中配置 GPU

const { app } = require('electron')

// 启用 GPU 加速
app.commandLine.appendSwitch('enable-gpu-rasterization')
app.commandLine.appendSwitch('enable-zero-copy')

// 禁用 GPU (调试用)
// app.commandLine.appendSwitch('disable-gpu')
// app.commandLine.appendSwitch('disable-software-rasterization')

// 选择 ANGLE 后端
// Windows: d3d11, d3d9, gl, vulkan
// macOS: metal, gl
// Linux: gl, vulkan
app.commandLine.appendSwitch('use-angle', 'gl')

// 忽略 GPU 黑名单
app.commandLine.appendSwitch('ignore-gpu-blacklist')

// 查看 GPU 信息
app.getGPUInfo('complete').then(info => {
  console.log('GPU Info:', info)
})
```
