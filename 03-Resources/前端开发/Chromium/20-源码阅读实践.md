### 本地搜索工具

```bash
# grep / ripgrep (rg)

# 基本搜索
rg "class Browser" --type cpp

# 搜索特定目录
rg "RenderFrame" third_party/blink/

# 搜索函数定义
rg "void.*Navigate\(" --type cpp

# 显示上下文
rg -C 5 "content::WebContents"

# 只显示文件名
rg -l "mojom"

# 使用正则
rg "class.*Impl.*:" --type cpp


# Git 搜索

# 搜索提交历史
git log -S "function_name" --oneline

# 搜索代码变更
git log -p -S "class_name"

# 按作者搜索
git log --author="name" --oneline

# 搜索文件历史
git log --follow -- path/to/file.cc
```

### 追踪方法

```bash
# 方法 1: 使用日志

# 添加 DVLOG 日志
DVLOG(1) << "Function called with param=" << param;

# 运行时启用日志
--v=1 --vmodule=*navigation*=2

# 方法 2: 使用调试器

# 在关键函数设置断点
(gdb) break content::WebContents::Navigate
(gdb) commands
> bt
> continue
> end

# 方法 3: 使用 tracing

# 添加 TRACE_EVENT
TRACE_EVENT0("category", "FunctionName");
TRACE_EVENT1("category", "FunctionName", "param", value);
```

```markdown
# 设计文档结构

## Summary / Overview

## Motivation
为什么要做这个功能，解决什么问题

## Goals / Non-Goals
目标和非目标，明确边界

## Design
详细设计，包括:
- 架构图
- API 设计
- 数据结构
- 流程图

## Testing
```
