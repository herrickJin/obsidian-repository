## 🧵 线程设计理念

### 为什么需要多线程？

```
单线程问题:
│
├── 阻塞操作导致 UI 无响应
├── 无法充分利用多核 CPU
└── 复杂任务影响整体性能

Chromium 多线程设计:
│
├── 响应性
│   └── UI 线程不被阻塞
│
├── 并行性
│   └── 多核 CPU 利用
│
├── 模块化
│   └── 按功能划分线程
│
└── 安全性
    └── 数据隔离减少竞争
```

## 🔄 消息循环 (Message Loop)

### 核心概念

```
Message Loop 工作原理:

┌─────────────────────────────────────────────────────────────┐
│                     Message Loop                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Task Queue (任务队列)               │   │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐          │   │
│  │  │Task1│ │Task2│ │Task3│ │Task4│ │ ... │          │   │
│  │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘          │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   Loop (循环)                        │   │
│  │                                                     │   │
│  │   while (running) {                                 │   │
│  │     task = queue.pop()                              │   │
│  │     task.execute()                                  │   │
│  │   }                                                 │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Deferred Tasks (延迟任务)               │   │
│  │  ├── Timer Tasks (定时器)                           │   │
│  │  └── Idle Tasks (空闲任务)                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 任务类型

```cpp
// Chromium 中的任务类型

// 1. 立即执行任务
task_runner->PostTask(FROM_HERE, base::BindOnce(&DoWork));

// 2. 延迟执行任务
task_runner->PostDelayedTask(FROM_HERE,
    base::BindOnce(&DoWorkLater),
    base::Seconds(5));

// 3. 指定时间执行任务
task_runner->PostTaskAt(time, base::BindOnce(&DoWork));

// 4. 空闲时执行任务
task_runner->PostIdleTask(base::BindOnce(&DoWorkWhenIdle));

// 5. 非嵌套任务
task_runner->PostNonNestableTask(FROM_HERE, base::BindOnce(&DoWork));
```

### JavaScript 中的类比

```javascript
// Chromium Message Loop 与 JavaScript Event Loop 的相似性

// Chromium: PostTask
task_runner->PostTask(FROM_HERE, base::BindOnce(&DoWork));

// JavaScript: setTimeout(0)
setTimeout(() => {
  doWork()
}, 0)

// Chromium: PostDelayedTask
task_runner->PostDelayedTask(FROM_HERE, base::BindOnce(&DoWork), base::Seconds(5));

// JavaScript: setTimeout
setTimeout(() => {
  doWork()
}, 5000)

// Chromium: PostIdleTask
task_runner->PostIdleTask(base::BindOnce(&DoWorkWhenIdle));

// JavaScript: requestIdleCallback
requestIdleCallback(() => {
  doWorkWhenIdle()
})
```

## 🧩 Task Runner

### Task Runner 类型

```
┌─────────────────────────────────────────────────────────────┐
│                    Task Runner 类型                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  SingleThreadTaskRunner                                     │
│  ├── 单线程执行                                              │
│  ├── 保证任务顺序                                            │
│  └── 用于需要线程亲和性的场景                                  │
│                                                             │
│  SequencedTaskRunner                                        │
│  ├── 保证任务顺序                                            │
│  ├── 可能使用线程池                                          │
│  └── 比单线程更高效                                          │
│                                                             │
│  ThreadPoolTaskRunner                                       │
│  ├── 线程池执行                                              │
│  ├── 任务可能并行                                            │
│  └── 用于独立任务                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 使用示例

```cpp
// 获取不同类型的 Task Runner

// 主线程 Task Runner
auto main_task_runner = base::SequencedTaskRunnerHandle::Get();

// IO 线程 Task Runner
auto io_task_runner = content::GetIOThreadTaskRunner({});

// 线程池 Task Runner
auto thread_pool_task_runner = base::ThreadPool::CreateTaskRunner(
    {base::TaskPriority::USER_VISIBLE});

// 后台任务 Runner
auto background_task_runner = base::ThreadPool::CreateTaskRunner(
    {base::TaskPriority::BEST_EFFORT});
```

## 🏭 线程池 (Thread Pool)

### 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                      Thread Pool                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Task Scheduler                      │   │
│  │  ├── 优先级排序                                      │   │
│  │  └── 任务分发                                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│           ┌──────────────┼──────────────┐                  │
│           ▼              ▼              ▼                  │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ Worker #1   │ │ Worker #2   │ │ Worker #N   │          │
│  │ (Foreground)│ │ (Foreground)│ │ (Background)│          │
│  └─────────────┘ └─────────────┘ └─────────────┘          │
│                                                             │
│  优先级队列:                                                 │
│  ├── USER_BLOCKING    - 最高优先级，阻塞用户                 │
│  ├── USER_VISIBLE     - 用户可见任务                        │
│  └── BEST_EFFORT      - 后台任务，最低优先级                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 任务优先级

```cpp
// 任务优先级示例

// 阻塞用户的任务 (如: 加载网页)
base::ThreadPool::PostTask(
    FROM_HERE,
    {base::TaskPriority::USER_BLOCKING},
    base::BindOnce(&LoadPage));

// 用户可见但非阻塞 (如: 预加载)
base::ThreadPool::PostTask(
    FROM_HERE,
    {base::TaskPriority::USER_VISIBLE},
    base::BindOnce(&PrefetchData));

// 后台任务 (如: 日志上传)
base::ThreadPool::PostTask(
    FROM_HERE,
    {base::TaskPriority::BEST_EFFORT},
    base::BindOnce(&UploadLogs));
```

## 🔒 线程安全与同步

### 同步原语

```
Chromium 同步原语
│
├── base::Lock
│   └── 互斥锁
│
├── base::ReadWriteLock
│   ├── 读锁 (共享)
│   └── 写锁 (独占)
│
├── base::ConditionVariable
│   └── 条件变量
│
├── base::WaitableEvent
│   └── 事件等待
│
├── base::AtomicRefCount
│   └── 原子引用计数
│
└── base::SequenceBound
    └── 序列边界类型
```

### 最佳实践

```cpp
// ❌ 避免: 手动加锁
base::AutoLock lock(lock_);
data_.push_back(item);

// ✅ 推荐: 使用 SequencedTaskRunner
class DataProcessor {
 public:
  DataProcessor()
      : task_runner_(base::ThreadPool::CreateSequencedTaskRunner({})) {}

  void ProcessItem(Item item) {
    task_runner_->PostTask(
        FROM_HERE,
        base::BindOnce(&DataProcessor::ProcessItemOnSequence,
                       base::Unretained(this), std::move(item)));
  }

 private:
  void ProcessItemOnSequence(Item item) {
    // 保证在同一个序列执行，无需加锁
    data_.push_back(std::move(item));
  }

  scoped_refptr<base::SequencedTaskRunner> task_runner_;
  std::vector<Item> data_;  // 无需锁保护
};
```

### 跨线程访问

```cpp
// 使用 base::ThreadChecker 检查线程

class MyClass {
 public:
  void DoSomething() {
    DCHECK_CALLED_ON_VALID_THREAD(thread_checker_);
    // 确保在正确的线程调用
  }

 private:
  THREAD_CHECKER(thread_checker_);
};

// 使用 WeakPtr 跨线程安全访问
class MyObject : public base::SupportsWeakPtr<MyObject> {
 public:
  void ScheduleWork() {
    base::ThreadPool::PostTask(
        FROM_HERE,
        base::BindOnce(&MyObject::DoWork, GetWeakPtr()));
  }

  void DoWork() {
    // 如果对象已销毁，不会调用
  }
};
```

### 主线程任务调度

```javascript
// 主线程任务优先级 (从高到低)

// 1. 输入处理 (最高优先级)
//    - 鼠标/触摸事件
//    - 键盘输入
//    - 滚动事件

// 2. 合成任务
//    - 滚动位置更新
//    - 动画帧

// 3. 默认任务
//    - 定时器
//    - 网络响应
//    - DOM 操作

// 4. 空闲任务 (最低优先级)
//    - 非关键渲染
//    - 垃圾回收标记

// 使用 requestIdleCallback
requestIdleCallback((deadline) => {
  while (deadline.timeRemaining() > 0) {
    // 执行低优先级任务
  }
})
```

## 📊 性能监控

### 线程性能指标

```javascript
// Performance API 监控

// 长任务检测
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log('Long Task:', {
      duration: entry.duration,
      name: entry.name,
      startTime: entry.startTime
    })
  }
})
observer.observe({ entryTypes: ['longtask'] })

// 帧率监控
let lastTime = performance.now()
let frames = 0

function measureFPS() {
  frames++
  const now = performance.now()
  if (now - lastTime >= 1000) {
    console.log('FPS:', frames)
    frames = 0
    lastTime = now
  }
  requestAnimationFrame(measureFPS)
}
requestAnimationFrame(measureFPS)
```
