# Chromium 多进程架构

## 🎯 学习目标

- 理解 Chromium 为什么采用多进程架构

- 了解进程间的协作关系
- 理解 Electron 中的进程映射

---

## 🏗️ 架构概览

### 为什么是多进程？

```
传统单进程浏览器问题:
│
├── 稳定性问题
│   └── 一个标签页崩溃 = 整个浏览器崩溃
│
├── 安全问题
│   └── 恶意页面可以访问所有数据
│
└── 性能问题
    └── 一个页面卡顿 = 整个浏览器卡顿

Chromium 多进程解决方案:
│
├── 进程隔离
│   └── 每个标签页独立进程
│
├── 沙箱保护
│   └── 限制页面进程权限
│
└── 资源管理
    └── 可独立回收问题进程
```

## 📦 进程类型详解

### 1. Browser Process (浏览器进程)

**职责**: 主控制进程，管理整个浏览器

```
Browser Process 职责
│
├── UI 管理
│   ├── 窗口和标签页
│   ├── 地址栏和工具栏
│   └── 用户交互事件
│
├── 网络管理
│   ├── 网络请求调度
│   ├── Cookie 管理
│   └── 证书处理
│
├── 存储管理
│   ├── 数据库
│   ├── 缓存
│   └── LocalStorage
│
├── 进程管理
│   ├── 创建/销毁渲染进程
│   ├── 进程间通信协调
│   └── 资源分配
│
└── 安全策略
    ├── 沙箱管理
    └── 权限控制
```

**线程模型**:

```
Browser Process 内部线程
│
├── Main Thread (主线程)
│   ├── UI 更新
│   ├── 用户输入处理
│   └── IPC 消息处理
│
├── IO Thread (IO 线程)
│   ├── 网络请求
│   ├── IPC 通信
│   └── 文件操作
│
├── File Thread (文件线程)
│   └── 文件系统操作
│
├── Cache Thread (缓存线程)
│   └── 缓存读写
│
└── DB Thread (数据库线程)
    └── 数据库操作
```

### 2. Renderer Process (渲染进程)

**职责**: 负责网页的解析、布局、渲染和 JavaScript 执行

```
Renderer Process 内部结构
│
├── Main Thread (主线程)
│   ├── HTML 解析 → DOM Tree
│   ├── CSS 解析 → CSSOM
│   ├── 样式计算
│   ├── 布局 (Layout)
│   ├── 绘制 (Paint)
│   └── JavaScript 执行 (V8)
│
├── Compositor Thread (合成线程)
│   ├── 图层合成
│   ├── 滚动处理
│   └── 动画处理
│
├── Raster Thread (光栅化线程)
│   └── 图层光栅化
│
└── Worker Threads
    ├── Web Workers
    └── Service Workers
```

**进程隔离策略**:

```
┌─────────────────────────────────────────────────────────────┐
│                    Site Isolation (站点隔离)                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  同源策略: 相同源的页面可能共享进程                            │
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │ Renderer #1     │  │ Renderer #2     │                  │
│  │ example.com     │  │ other.com       │                  │
│  │ example.com/a   │  │ malicious.com   │                  │
│  │ example.com/b   │  │                 │                  │
│  └─────────────────┘  └─────────────────┘                  │
│                                                             │
│  不同源站点使用独立进程，防止数据泄露                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. GPU Process (GPU 进程)

**职责**: 处理 GPU 相关任务

```
GPU Process 职责
│
├── 图形渲染
│   ├── OpenGL / Vulkan / Metal
│   ├── 合成器
│   └── WebGL
│
├── 视频处理
│   ├── 硬件解码
│   └── 视频后处理
│
└── GPU 命令缓冲区
    ├── 接收渲染命令
    └── 提交到 GPU
```

**GPU 进程架构**:

```
┌─────────────────────────────────────────────────────────────┐
│                      GPU Process                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Renderer Processes                                         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐                      │
│  │ GL 命令  │ │ GL 命令  │ │ GL 命令  │                      │
│  └────┬────┘ └────┬────┘ └────┬────┘                      │
│       │           │           │                            │
│       ▼           ▼           ▼                            │
│  ┌─────────────────────────────────────┐                   │
│  │         Command Buffer              │                   │
│  │       (命令缓冲区/共享内存)           │                   │
│  └─────────────────┬───────────────────┘                   │
│                    │                                        │
│                    ▼                                        │
│  ┌─────────────────────────────────────┐                   │
│  │           GPU Process               │                   │
│  │  ┌─────────────────────────────┐   │                   │
│  │  │    Command Processor        │   │                   │
│  │  └─────────────┬───────────────┘   │                   │
│  │                │                    │                   │
│  │                ▼                    │                   │
│  │  ┌─────────────────────────────┐   │                   │
│  │  │   GPU Driver (OpenGL/Vulkan)│   │                   │
│  │  └─────────────────────────────┘   │                   │
│  └─────────────────────────────────────┘                   │
│                    │                                        │
│                    ▼                                        │
│               硬件 GPU                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4. Utility Process (实用进程)

**职责**: 处理通用任务，提供沙箱隔离

```
Utility Process 用途
│
├── 媒体处理
│   ├── 音频解码
│   ├── 视频解码
│   └── 格式转换
│
├── 文件处理
│   ├── 文件解压
│   ├── PDF 渲染
│   └── 文件预览
│
├── 网络服务
│   ├── 代理解析
│   └── 证书验证
│
└── 扩展服务
    └── 扩展隔离运行
```

### 5. Plugin Process (插件进程)

**职责**: 运行浏览器插件 (如 PDF 阅读器)

```
Plugin Process
│
├── PPAPI 插件
│   ├── PDF 阅读器
│   └── Flash (已废弃)
│
├── Native Messaging
│   └── 与本地应用通信
│
└── 独立沙箱
    └── 插件崩溃不影响浏览器
```

### 6. Extension Process (扩展进程)

**职责**: 运行浏览器扩展

```
Extension Process
│
├── Background Script
│   └── 扩展后台逻辑
│
├── Content Script
│   └── 注入页面的脚本 (在渲染进程中)
│
└── Popup/Options Page
    └── 扩展 UI 页面
```

## 🔧 进程管理实践

### 查看进程信息

```javascript
// Chrome 中查看进程
// 1. 打开 Chrome Task Manager: Shift + Esc
// 2. 访问 chrome://process-internals

// Electron 中查看进程
const { app } = require('electron')

app.getAppMetrics().forEach(metric => {
  console.log(`PID: ${metric.pid}, Type: ${metric.type}`)
  console.log(`Memory: ${metric.memory.workingSetSize / 1024 / 1024} MB`)
})
```

### 进程资源限制

```javascript
// Electron 中限制渲染进程数量
const { app } = require('electron')

// 设置最大渲染进程数
app.commandLine.appendSwitch('renderer-process-limit', '4')

// 设置进程内存限制
app.commandLine.appendSwitch('max-old-space-size', '512')

// 禁用站点隔离 (不推荐)
// app.commandLine.appendSwitch('disable-site-isolation-trials')
```
